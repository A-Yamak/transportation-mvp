üé® Code Quality Standards (MANDATORY)

PSR-12 Code Style

4 spaces indentation, proper spacing, max 120 chars per line. Run ./vendor/bin/phpcs to verify.

Type Safety (PHP 8.4)

declare(strict_types=1) on all files, type hints on ALL parameters, return type declarations, no mixed types.

PHPDoc Documentation

Class-level and method-level doc blocks with @param, @return, @throws. Inline comments for complex logic.

Clean Code Principles

Descriptive variable names, Single Responsibility, DRY, no magic numbers, use constants/config.

Testing (TDD Approach)

Write tests BEFORE implementation when possible. Feature tests for endpoints, 80%+ coverage target.

Laravel Best Practices

Form Requests for validation, API Resources for responses, dependency injection, Eloquent relationships.

Security

Validate ALL user input, use authorization (policies), sanitize output, no SQL injection or XSS.

No Debug Code

No dd(), var_dump(), console.log in production code. No commented-out code in commits.
